# analog — a non-standard-cell layout testcase

Geometry-only fixture proving the `vyges-layout` kernel has **zero digital
assumptions**: it is pure layers/datatypes/polygons, so an analog block round-trips
and booleans exactly as a digital one does.

`analog.gds` (generated by `cargo run --example gen_analog`) is an `analog` cell with
the shapes an analog designer actually draws — *not* a standard cell:

- **guard ring** (layer 67/20, metal1) — a rectilinear *ring* (donut) polygon enclosing
  the device, the canonical analog isolation structure.
- **MIM capacitor** — a bottom plate (65/20) and a top plate (66/20) on two different
  layers that **overlap**; their intersection is the capacitor area.
- **resistor snake** (layer 64/20, poly) — a serpentine (S-shaped) rectilinear polygon,
  the way a poly resistor is laid out to pack length into area.

The test `tests/analog.rs` exercises:

- **GDS write → read round-trip equality** — every shape (and its exact vertices)
  survives `to_bytes`/`parse`.
- a **polygon boolean across two layers** — top-plate (66/20) `AND` bottom-plate
  (65/20) returns the exact overlap rectangle (the MIM cap area), and `OR` of the two
  guard-ring halves reunites the full ring area.
